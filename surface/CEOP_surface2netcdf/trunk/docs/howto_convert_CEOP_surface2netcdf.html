Purpose

This document contains instructions for converting CEOP Surface Meteorological and Radiation (.sfc) data files from the CEOP Surface ASCII format into the CEOP Surface netCDF format and converting the CEOP Surface netCDF format back to the CEOP Surface ASCII format.

Description


The CEOP Surface Meteorological and Radiation format conversion programs were written using Java 6 and Unidata's NetCDF-Java library version 4.2. The Unidata NetCDF-Java library implements a Common Data Model (CDM) which is an abstract data model for scientific datasets. The necessary Unidata jar files are netcdfUI-4.2.jar and toolsUI-4.2.jar. The software uses the getopt.jar file, version 1.0.8, which is a Java port of GNU getopt. GNU getopt is a class for parsing command line arguments passed to programs based on the C getopt functions in glibc 2.0.6. The software uses the utilities.jar file which contain general utilities for unit conversions written by DMG personnel. The conversion programs were written, compiled, and tested using the NetBeans 7.0.1 interface.

The conversion program sfc2cdf/CEOPsfciosp, which is the conversion from the CEOP Surface Meteorological and Radiation format to netCDF, utilizes the Input/Output Service Provider (IOSP) interface from Unidata. The output netCDF files were created using the Climate and Forecast (CF) Metadata Conventions version 1.6-beta. The CF Conventions version 1.6 had not been formally accepted when these CEOP conversions were run, but some of the features in the 1.6 version of the CF Conventions were necessary for creating the netCDF files, specifically the Timeseries of Station Data. The CF Conventions version 1.6 was formally accepted in December 2011 and the Timeseries of Station Data has been superseded by the Discrete Sampling Geometries. The CF Standard Name Table was used in creating the variable names in the netCDF file. The netCDF data files were run through the CF Compliance checker to verify that the files correctly met the CF Conventions as of August 2011.

CEOP ASCII data files may contain more than one station, but each station is written to a separate netCDF file, therefore there may be many netCDF files created from one CEOP ASCII data file.

The conversion program cdf2sfc/CEOPcdf2sfc converts the CEOP Surface Meteorological and Radiation netCDF format back to the original ASCII format. This conversion was used to verify that the original conversions to the netCDF format were created correctly. The original CEOP Surface ASCII format files were compared to the CEOP Surface ASCII format files created from the netCDF format files.

Running the Conversion Program


The program is run using the jar file contained in the dist directory. This directory contains the jarfile and the necessary libraries for running the software. The command to run the conversion from CEOP sfc ASCII data files to CEOP sfc netCDF data files is:

java -jar CEOPsfciosp.jar -i <inCEOPfile> -o <outdirname>

<inCEOPfile> is replaced with the name of the CEOP sfc ASCII data file to convert.
<outdirname> is replaced with the name of the directory where the CEOP sfc netCDF data files will be written.
The -o <outdirname> is an optional parameter and will default to the current directory.

The output files created will have the same file name, without the dates and with the station name added, as the input file with _sfc.nc as the last part of the file name instead of .sfc.

Example: java -jar CEOPsfciosp LBA_Santarem_20021001_20030930.sfc -o conversionDirectory
   will create conversionDirectory/LBA_Santarem_ForestKm83_sfc.nc

The command to run the conversion from CEOP sfc netCDF data files to CEOP sfc ASCII data files is:

java -jar CEOPcdf2sfc.jar -i <inCEOPfile> -o <outdirname>

<inCEOPfile> is replaced with the name of the CEOP sfc netCDF data file to convert.
<outdirname> is replaced with the name of the directory where the CEOP sfc ASCII data files will be written.
The -o <outdirname> is an optional parameter and will default to the current directory.

The output files created will have the same file name, with the dates included, as the input file with _nc.sfc as the last part of the file name instead of _sfc.nc.

Example: java -jar CEOPcdf2sfc LBA_Santarem_ForestKm83_sfc.nc -o outputDirectory
   will create outputDirectory/LBA_Santarem_ForestKm83_20021001_20030930_nc.sfc
